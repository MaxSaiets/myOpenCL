# Дизайн модуля планування

## Мета
Створити гнучкий, розширюваний модуль планування для роботи зі складними багатокроковими задачами.

## Вимоги
- Підтримка декомпозиції цілей
- Управління залежностями між підзадачами
- Адаптивність до змінних умов (зміни в оточенні, новий зворотний зв'язок)
- Інтеграція з іншими модулями (пам'ять, інструменти, комунікація)
- Відновлення стану після перерв

## Архітектура
- `Planner`: основний клас, що керує процесом планування
- `Goal`: представлення цілі з метою, пріоритетом, станом
- `Task`: підзадача з описом, залежностями, статусом
- `Plan`: впорядкований список задач з графом залежностей

## Інтеграції
- MEMORY.md → збереження стану плану та ключових рішень
- canvas → візуалізація графу плану
- tools → автоматичне виконання дій
- message → звітування про стан та запити на підтвердження
