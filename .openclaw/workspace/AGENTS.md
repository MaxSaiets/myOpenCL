# AGENTS.md — Операційні інструкції для Claw

## Startup
Читай USER.md і SOUL.md. Виконуй задачу — не питай.

---

## ГОЛОВНЕ ПРАВИЛО: завжди відповідай через tool `message`

❌ Заборонено: повертати текст як фінальну відповідь  
✅ Обов'язково: кожна відповідь — через tool `message`

**Схема кожної задачі:**
1. `message` → «Роблю: [що]»
2. Виконуй tools (exec / read / write)
3. `message` → «✅ Готово! [докази]» або «❌ [помилка]»

**Крок 3 ОБОВ'ЯЗКОВИЙ навіть для тривіальних задач.**

---

## КРИТИЧНО: після `message` → продовжуй роботу

Якщо tool `message` повернув `{"ok": true}` — це лише підтвердження доставки.
**Після цього ОБОВ'ЯЗКОВО виклич наступний tool (exec, read, тощо).**

❌ Заборонено: зупинятись після `{"ok": true}` від message  
✅ Правильно: message("Роблю...") → exec("команда") → message("✅ Готово!")

---

## Стоп-правило

- ❌ Не крутись у циклі помилок без повідомлення
- ❌ Не пробуй одне й те саме 3+ рази
- ❌ Не мовчи якщо задача не йде 5+ хвилин
- Помилка після 2 спроб → повідом через message: «❌ [причина]. Жду команди.»

---

## Кілька команд — один exec

```
exec: cmd1 && cmd2 && cmd3
```
Потім один `message` з усіма результатами.

❌ Не розбивай на окремі exec-виклики.

---

## Запис файлів

Використовуй `exec` з bash замість tool `write`:
```
exec: printf 'line1\nline2\n' > /path/file
exec: cat << 'EOF' > /path/file
content
EOF
```

## Верифікація

Після кожної дії — перевіряй результат:

| Дія | Перевірка |
|-----|-----------|
| файл створено | `ls -la <path>` |
| git push | `git log --oneline -1` |
| pm2 restart | `pm2 status` |

---

## Інструменти

- `exec` — bash команда (host: gateway)
- `read` — читати файл (вказуй `path`)
- `write` — записати файл (може не працювати — використовуй exec)
- `message` — надіслати в Telegram (ЄДИНИЙ спосіб відповісти!)
